¿Qué vamos a hacer?

1) Conectase a la API de Marvel.

2) Obtener los comics que saldrán semana que viene.

El formato de Comics de la tienda de Stuart es:

Un Comic tiene: - Título - ThumbailUrl - Precio
¿Cuál es el objectivo?

1) Practicar la conexión a servicios de terceros

2) Ver los problemas que aparecen al conectarnos a servicios externos.

3) Después de tener la primera versión puede que haya sorpresas...

¿Cómo podemos hacer esto?

Conexión desde Servidor
Url: http://gateway.marvel.com:80/v1/public/comics?dateDescriptor=nextWeek&ts=XXX&apikey=XXX&hash=XXX

ts = timestamp o cualquier número.
apiKey = public key
hash = md5(ts+privateKey+publicKey)

Conexión desde cliente:
Url: http://gateway.marvel.com:80/v1/public/comics?dateDescriptor=nextWeek&apikey=XXXX

Discusión

¿Qué le pasará a la aplicación si la API falla?
¿Que le pasa a nuestro código si cambian el formato?
¿Que pasa a nustro código si cambian la llamada?
¿qué patrones nos pueden ayudar?
Patrones que nos pueden ayudar:

Respository: Para el acceso a datos, podemos cambiar entre nuestro sistema y la API fácilmente.
Gateway: para simplificar el acceso a la llamada de servicios externos.
Mapper o Builder Pattern: Transformar la respuesta a nuestro formato.
